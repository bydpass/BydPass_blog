# 信息收集仓库（BydPass_blog）

这是一个用于集中收集、归档、整理和复用信息的个人知识仓库。

## 目标

将零散收藏转化为可检索、可维护、可复用的 Markdown 信息系统。

未来可接入 LLM librarian、自动整理脚本、Obsidian 联动等工具，当前阶段先用最简单的方式跑通流程。

## 当前用途

- 小红书收藏整理
- 商品与工具种草记录
- 教程与文章归档
- 灵感与想法沉淀
- 为未来 LLM librarian 预留结构

## 工作流

1. 新信息先进入 `inbox/YYYY-MM.md`，按 inbox 模板填写
2. 每隔一段时间批量整理
3. 有价值内容移动到 `notes/` 对应分类下，按正式笔记模板重写
4. 图片、PDF、附件放入 `assets/`
5. 过时或废弃内容放入 `archive/`

### inbox 条目模板

每条新信息先在 `inbox/YYYY-MM.md` 中按此格式记录：

```md
## 标题

- 来源：
- 链接：
- 类型：
- 标签：
- 收集日期：
- 一句话价值：
- 原始摘录：
- 我的判断：
- 后续动作：
```

### 正式笔记模板

inbox 条目被整理进 `notes/` 后，使用此格式：

```md
# 标题

## 基本信息

- 来源：
- 链接：
- 类型：
- 标签：
- 创建日期：
- 最后更新：

## 核心价值

一句话说明这个条目为什么值得保存。

## 内容摘要

用自己的话压缩原始信息。

## 我的判断

记录个人判断，不只复制原文。

## 使用场景

说明未来什么时候会用到它。

## 后续动作

- [ ]
- [ ]

## 相关材料

-
```

## 目录结构

```text
BydPass_blog/
├── README.md           本文件
├── tags.md             标签系统说明
├── inbox/              临时收集箱，按月份分文件
│   └── 2026-05.md
├── notes/              整理后的正式笔记
│   ├── products/       商品、种草、消费决策
│   ├── tools/          软件、网站、工具、工作流
│   ├── ideas/          灵感、观点、可发展想法
│   ├── tutorials/      教程、操作指南、学习资料
│   └── references/     参考资料、文章、论文、背景材料
├── assets/             素材区
│   ├── images/         图片、截图
│   └── files/          PDF、文档、附件
└── archive/            归档区（过时内容、废弃方案、v0.1 spec）
```

### 素材命名建议

```text
YYYY-MM-DD-short-description.ext
```

例：

```text
2026-05-17-xhs-keyboard-layout.png
2026-05-17-github-guide.pdf
```
