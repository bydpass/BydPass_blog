# ROADMAP

## Current Version

v0.1 - GitHub Markdown 信息收集仓库骨架

## Current Goal

先手动跑通 20 条真实信息收集流程，验证 inbox / notes / tags / assets / archive 是否够用。

## v0.1 Rules

- 新信息先进入 `inbox/`
- 不急着分类
- 不写脚本
- 不接自动化
- 不装复杂插件
- 不引入数据库
- 不把 repo 变成工程项目

## Next Test

- [ ] 5 条工具 / 网站

## Review Questions

录入 20 条后复盘：

1. inbox 模板是否太重？
2. tags 是否够用？
3. notes 五分类是否合理？
4. assets 命名是否麻烦？
5. Obsidian 打开后是否顺手？
6. GitHub 网页端编辑是否顺手？
7. 是否需要模板文件夹？
8. 是否需要 LLM librarian prompt？
